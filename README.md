# Project BI MVP

一个可部署的经营管理系统 MVP，用于验证“商机/项目基础执行数据 -> 经营预测 -> KPI -> 调度闭环 -> 分角色看板”的主链路。

## 功能范围

- 分角色看板：总经理、经营管理人员、业务组总监、售前人员、项目经理。
- 商机管理：阶段、概率、预计金额、预计签约、下一步动作、风险。
- 项目管理：阶段、进度、健康度、收入、成本、毛利、回款、应收。
- 经营预测：预测收入、预测成本、预测毛利、预测回款、资源缺口、审核状态。
- KPI 跟踪：目标、实际、完成率、差距。
- 调度动作：来源、责任人、优先级、截止日期、状态、进展反馈。
- 权限演示：总经理/经营管理看全部，业务组总监看本组，售前看本人商机，项目经理看本人项目。
- 用户管理：维护用户、角色、组织、状态、邮箱。
- 组织管理：维护部门、业务组和上下级关系。
- 权限管理：查看和维护角色权限矩阵，自动生成默认数据范围。

## 权限模型

角色不是固定身份，而是一组可配置能力：

- 可用功能：例如查看商机、维护项目、审核预测、管理调度。
- 可见数据：例如全部数据、本业务组、本人商机、本人项目。
- 可进入页面：例如经营驾驶舱、商机管理、项目中心、用户管理、组织管理、权限管理。

用户只负责绑定角色和组织；系统根据角色拥有的页面权限显示导航，根据角色的数据范围过滤业务数据。

## 本地启动

```bash
python3 app.py
```

打开：

```text
http://127.0.0.1:8000
```

如果 8000 端口被占用，可以指定端口：

```bash
PORT=8010 python3 app.py
```

然后打开：

```text
http://127.0.0.1:8010
```

首次启动会自动创建 SQLite 数据库并写入演示数据：

```text
data/mvp.sqlite
```

如需重置演示数据，停止服务后删除 `data/mvp.sqlite`，再重新启动。

## Docker 部署

构建镜像：

```bash
docker build -t project-bi-mvp .
```

运行：

```bash
docker run --rm -p 8000:8000 project-bi-mvp
```

## 技术说明

- 后端：Python 标准库 `http.server` + `sqlite3`，无外部依赖。
- 前端：原生 HTML/CSS/JavaScript 单页应用。
- 数据库：SQLite。
- 部署：可直接 Python 运行，也可用 Docker 容器运行。

这是 MVP 骨架，后续可以替换为 FastAPI/NestJS + PostgreSQL + React/Ant Design Pro，但当前版本已经能验证核心业务对象、权限边界和经营管理闭环。
