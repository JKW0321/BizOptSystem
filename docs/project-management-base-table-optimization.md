# 项目管理底表优化方案

## 1. 核心判断

当前 `项目管理底表.xlsx` 的问题不是字段不够，而是管理对象混在一起：

- 项目主数据、合同预算、年度计划、月度收入、成本、毛利、回款预测都横向堆在同一张表。
- 字段偏经营汇总，缺少软件研发和交付项目必需的过程管理字段。
- 无法清楚回答“项目现在处于什么阶段、卡在哪里、谁负责、下一里程碑是什么、交付风险是什么”。
- 1-12 月收入/支出/毛利横向展开，适合临时 Excel 汇总，不适合持续维护和系统化。

建议把底表从“经营指标填报表”改造成“项目全过程管理底座”。经营指标仍然保留，但作为项目数据的结果，而不是项目管理的全部。

## 2. 推荐管理模型

按照通用软件研发和交付项目，建议采用以下生命周期：

1. 商机/售前。
2. 立项。
3. 项目计划。
4. 需求分析。
5. 方案/设计。
6. 开发/配置/集成。
7. 测试/联调。
8. 上线/试运行。
9. 验收。
10. 运维/质保。
11. 关闭。

不同类型项目可以裁剪：

- 自研产品研发：更重需求、版本、研发迭代、测试、发布。
- 客户交付项目：更重合同范围、交付里程碑、客户验收、回款。
- 系统集成项目：更重设备/第三方供应商、到货、安装、联调。
- 运维项目：更重 SLA、工单、服务周期、续约。
- 咨询规划项目：更重成果物、评审、验收。

## 3. 底表设计原则

### 3.1 从“一张宽表”改为“多张窄表”

建议至少拆成 10 张表：

| 表 | 管理对象 | 用途 |
| --- | --- | --- |
| `客户主表` | 一个客户一行 | 客户身份、行业、区域、客户级别、归属关系 |
| `项目主表` | 一个项目一行 | 项目身份、分类、负责人、状态 |
| `项目阶段里程碑表` | 一个里程碑一行 | 计划、进度、验收节点 |
| `需求与范围表` | 一个需求/范围项一行 | 范围、变更、交付物管理 |
| `项目资源表` | 一个人员投入一行 | 自有/外包资源、投入周期、人月 |
| `项目预算表` | 一个成本/收入科目一行 | 预算、计划、目标毛利 |
| `月度经营计划表` | 一个项目一个月份一行 | 月度收入、成本、毛利、回款预测 |
| `月度经营实际表` | 一个项目一个月份一行 | 实际收入、成本、毛利、回款 |
| `风险问题表` | 一个风险/问题一行 | 风险闭环、责任人、解决期限 |
| `变更与验收表` | 一个变更/验收项一行 | 范围变更、验收资料、结项依据 |

如果第一期要更轻，可以先做 5 张：

- `客户主表`
- `项目主表`
- `项目阶段里程碑表`
- `项目资源表`
- `月度经营计划表`
- `风险问题表`

经营实际数可以先从财务/经营 Excel 导入，不要求项目经理维护。

### 3.2 区分三类字段

| 类型 | 来源 | 是否项目经理维护 |
| --- | --- | --- |
| 项目管理字段 | 项目经理/业务组总监/经营管理人员 | 是 |
| 经营计划字段 | 项目经理填报，业务组总监审核 | 是，但需留痕 |
| 经营实际字段 | 财务/经营导入 | 否，项目经理只补充说明 |

这样能避免“项目经理手工改实际经营数”，也能保留经营管理的可信口径。

## 4. 优化后的底表结构

## 4.1 客户主表

一个客户一行，回答“这个客户是谁、归属谁、贡献和风险如何”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 客户编号 | 是 | 客户唯一编码，优先沿用现有客户编号 |
| 客户名称 | 是 | 标准客户名称 |
| 客户简称 | 否 | 看板展示用 |
| 统一社会信用代码 | 否 | 外部客户建议维护，用于去重 |
| 客户类型 | 是 | 外部客户、内部单位、合作伙伴、集团客户 |
| 客户级别 | 否 | 战略、重点、普通、低活跃 |
| 所属行业 | 否 | 行业分析 |
| 所在省份/区域 | 否 | 区域分析 |
| 客户经理 | 否 | 销售或客户责任人 |
| 归属部门/业务组 | 否 | 客户经营归属 |
| 合作状态 | 是 | 潜在、合作中、暂停、终止 |
| 首次合作日期 | 否 | 历史分析 |
| 最近合作日期 | 否 | 活跃度分析 |
| 累计签约金额 | 否 | 系统从合同汇总 |
| 累计收入 | 否 | 系统从实际收入汇总 |
| 累计回款 | 否 | 系统从回款汇总 |
| 应收余额 | 否 | 系统从应收汇总 |
| 客户风险等级 | 否 | 可由逾期应收、验收争议、合同风险生成 |
| 备注 | 否 | 补充说明 |

其中累计签约、累计收入、累计回款、应收余额不建议手工填，应该由项目、合同、回款和应收自动汇总。

## 4.2 项目主表

一个项目一行，回答“这是什么项目、谁负责、现在什么状态”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 唯一编码，对接财务/合同/计收数据 |
| 项目名称 | 是 | 标准项目名称 |
| 项目简称 | 否 | 看板展示用 |
| 客户编号 | 是 | 关联客户主表 |
| 客户名称 | 是 | 外部客户或内部委托方，建议从客户主表带出 |
| 客户行业 | 否 | 行业分析 |
| 客户省份/区域 | 否 | 区域分析 |
| 内外部 | 是 | 内部/外部 |
| 项目类型 | 是 | 自研研发、客户交付、系统集成、运维、咨询、外包服务 |
| 交付模式 | 是 | 自主实施、联合交付、第三方转包、省分派单 |
| 产品线 | 是 | ERP、供应链、财务、人力、数字化等 |
| 所属部门 | 是 | 部门 |
| 所属业务组 | 是 | 业务组 |
| 项目经理 | 是 | 项目第一责任人 |
| 销售经理 | 否 | 商机/合同协同责任人 |
| 技术负责人 | 否 | 技术交付责任人 |
| 立项日期 | 是 | 项目正式立项日期 |
| 计划开始日期 | 是 | 项目计划开始 |
| 计划结束日期 | 是 | 项目计划结束 |
| 实际开始日期 | 否 | 实际启动 |
| 实际结束日期 | 否 | 实际完成 |
| 项目状态 | 是 | 商机中、待立项、已立项、交付中、验收中、运维中、已关闭、暂停、终止 |
| 当前阶段 | 是 | 需求、设计、开发、测试、上线、验收等 |
| 进度百分比 | 是 | 0-100%，建议由里程碑自动汇总 |
| 健康状态 | 是 | 绿、黄、红 |
| 是否重点项目 | 否 | 重点项目驾驶舱 |
| 是否存量项目 | 否 | 存量/增量 |
| 备注 | 否 | 补充说明 |

建议删除或下沉的字段：

- `1月收入` 到 `12月双计收入` 不应放在项目主表。
- `截止2025年底已计收收入`、`2026年累计计收` 等累计经营数不应手工维护，应由月度实际表汇总。

## 4.3 项目阶段里程碑表

一个项目可以有多个里程碑，回答“项目怎么交付、节点是否延期”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目主表 |
| 阶段序号 | 是 | 1、2、3... |
| 阶段名称 | 是 | 需求、设计、开发、测试、上线、验收 |
| 里程碑名称 | 是 | 如“需求确认”、“UAT通过”、“上线完成” |
| 交付物 | 是 | 需求规格说明书、设计文档、测试报告、上线报告、验收单 |
| 责任人 | 是 | 阶段负责人 |
| 计划开始日期 | 是 | 计划开始 |
| 计划完成日期 | 是 | 计划完成 |
| 实际开始日期 | 否 | 实际开始 |
| 实际完成日期 | 否 | 实际完成 |
| 完成比例 | 是 | 0-100% |
| 状态 | 是 | 未开始、进行中、已完成、延期、暂停、取消 |
| 是否客户确认 | 否 | 客户交付类项目建议必填 |
| 确认日期 | 否 | 客户/内部确认日期 |
| 延期原因 | 否 | 延期时必填 |
| 下一步动作 | 否 | 当前推进动作 |

项目进度建议由里程碑完成比例加权汇总，而不是项目经理直接拍一个进度数。

## 4.4 需求与范围表

软件项目最容易失控的是范围，建议单独管理。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目 |
| 需求编号 | 是 | 唯一编号 |
| 需求名称 | 是 | 需求/模块/功能点 |
| 需求类型 | 是 | 业务需求、功能需求、接口需求、数据需求、报表需求、非功能需求 |
| 来源 | 是 | 合同、客户新增、内部产品规划、监管要求 |
| 优先级 | 是 | 高、中、低 |
| 是否合同范围内 | 是 | 范围控制关键字段 |
| 预计工作量人日 | 否 | 估算 |
| 负责人 | 是 | 需求负责人 |
| 状态 | 是 | 待确认、已确认、设计中、开发中、测试中、已上线、已取消 |
| 关联里程碑 | 否 | 对应交付节点 |
| 验收标准 | 否 | 可验收描述 |
| 备注 | 否 | 补充 |

对于客户交付项目，建议把合同范围、需求确认和变更联动起来。否则后续毛利下降、延期、客户不验收，很难追溯。

## 4.5 项目资源表

回答“谁在项目上投入了多少资源，资源成本是多少”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目 |
| 月份 | 是 | yyyy-mm |
| 人员类型 | 是 | 自有、外包、供应商团队 |
| 人员姓名/角色 | 是 | 可填具体姓名或角色 |
| 所属部门/供应商 | 是 | 自有部门或外包供应商 |
| 岗位角色 | 是 | 项目经理、产品、架构、开发、测试、实施、运维 |
| 投入比例 | 是 | 如 50%、100% |
| 投入人月 | 是 | 用于成本核算 |
| 成本单价 | 否 | 可按权限隐藏 |
| 计划成本 | 否 | 投入人月 * 单价 |
| 实际成本 | 否 | 财务或人力导入 |
| 是否关键资源 | 否 | 关键人员预警 |
| 资源状态 | 是 | 计划、已确认、已投入、已释放 |

第一期可以先只维护“角色 + 人月”，不必维护到人。第二期再与人力或外包台账集成。

## 4.6 项目预算表

回答“项目立项时承诺的收入、成本、毛利是什么”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目 |
| 预算版本 | 是 | 初始版、调整版 V1/V2 |
| 收入合同额含税 | 是 | 合同或预计合同金额 |
| 收入合同额不含税 | 是 | 按税率计算 |
| 税率 | 是 | 多税率建议拆行 |
| 预算成本含税 | 是 | 项目总预算成本 |
| 预算成本不含税 | 是 | 不含税成本 |
| 自有资源成本 | 否 | 内部人力成本 |
| 外包成本 | 否 | 外包人员或服务 |
| 第三方采购成本 | 否 | 设备、软件、服务采购 |
| 分公司成本 | 否 | 省分/分公司成本 |
| 其他费用 | 否 | 差旅、云资源等 |
| 预算毛利 | 是 | 收入 - 成本 |
| 预算毛利率 | 是 | 预算毛利 / 收入 |
| 预算生效日期 | 是 | 版本生效日期 |
| 审批状态 | 是 | 草稿、待审核、已审核、驳回 |
| 调整原因 | 调整时必填 | 预算调整说明 |

不建议只保留一个“立项毛利率”。毛利率必须能向下穿透到成本构成，否则项目亏损时无法定位原因。

## 4.7 月度经营计划表

把原底表的 1-12 月收入/支出/毛利拆成纵向表。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目 |
| 年份 | 是 | 2026 |
| 月份 | 是 | 1-12 |
| 计划确认收入不含税 | 是 | 计收计划 |
| 计划管理口径收入 | 是 | 管理口径 |
| 计划成本不含税 | 是 | 成本计划 |
| 计划分摊前毛利 | 否 | 可公式生成 |
| 计划分摊后毛利 | 是 | 经营目标 |
| 计划双计收入 | 否 | 如需保留双计口径 |
| 计划回款含税 | 否 | 回款预测 |
| 预计账龄 | 否 | 预计回款时间 |
| 计划说明 | 否 | 关键假设 |
| 填报人 | 是 | 留痕 |
| 填报时间 | 是 | 留痕 |
| 审核状态 | 是 | 草稿、已提交、已审核 |

这样做的好处：

- 可以按月滚动调整。
- 可以直接对比计划与实际。
- 不需要每年新增 60 多个字段。

| 字段 | 是否必填 | 说明 |

| 年份 | 是 | 年份 |
| 月份 | 是 | 月份 |
| 合同编号 | 否 | 可穿透到合同 |
| 单据编号 | 否 | 可穿透到计收单据 |
| 确认收入不含税 | 是 | 实际收入 |
| 管理口径收入 | 是 | 管理口径 |
| 主营收入 | 否 | 主营收入标记 |
| 成本金额不含税 | 是 | 实际成本 |
| 分摊后毛利 | 是 | 实际毛利 |
| 双计收入 | 否 | 双计口径 |
| 回款金额含税 | 否 | 实际回款 |
| 应收余额含税 | 否 | 月末应收 |
| 数据来源 | 是 | Excel、财务系统、手工调整 |
| 导入批次 | 是 | 追溯来源 |

## 4.9 风险问题表

回答“项目哪里有风险，谁负责解决，何时关闭”。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目编号 | 是 | 关联项目 |
| 类型 | 是 | 风险、问题 |
| 风险/问题分类 | 是 | 进度、范围、成本、质量、资源、客户、供应商、回款、合同 |
| 标题 | 是 | 简短描述 |
| 详细描述 | 是 | 具体情况 |
| 影响程度 | 是 | 高、中、低 |
| 发生概率 | 风险必填 | 高、中、低 |
| 风险等级 | 是 | 可由影响 * 概率生成 |
| 责任人 | 是 | 处理责任人 |
| 应对措施 | 是 | 解决方案 |
| 计划关闭日期 | 是 | 截止时间 |
| 实际关闭日期 | 否 | 关闭时填写 |
| 状态 | 是 | 打开、处理中、已关闭、挂起 |
| 是否升级 | 否 | 是否需要业务组总监或经营管理人员介入 |

项目健康状态建议由风险、进度、毛利、回款共同计算：

- 红：关键里程碑延期、毛利率显著低于预算、重大回款风险、客户验收受阻。
- 黄：存在延期趋势、资源不足、成本超预算趋势。
- 绿：按计划推进，无重大经营风险。

## 5. 推荐字段精简：项目主表不要超过 35 个字段

项目主表建议保留核心身份和状态字段，不要承担所有经营明细。

建议项目主表字段：

1. 项目编号。
2. 项目名称。
3. 客户编号。
4. 客户名称。
5. 内外部。
6. 项目类型。
7. 交付模式。
8. 产品线。
9. 所属部门。
10. 所属业务组。
11. 项目经理。
12. 技术负责人。
13. 销售经理。

## 6. 原底表字段的处理建议

| 原字段类型 | 处理方式 |
| --- | --- |
| 客户名称、客户编号、统一社会信用代码、客户行业、客户省份 | 拆到客户主表，项目主表通过客户编号关联 |
| 项目编号、项目名称、产品线、项目经理、业务组、内外部 | 保留到项目主表 |
| 交付方式、项目类型、客户省份 | 保留并规范字典 |
| 收入合同额、项目总成本、立项毛利率 | 移到项目预算表，支持多版本 |
| 项目已回款、应收账款、未计收、累计计收 | 改为从经营实际表/应收表汇总 |
| 截止上年底、2026累计等字段 | 不手工维护，由月度实际表按时间汇总 |
| 1-12 月收入/支出/毛利/双计收入 | 拆成月度经营计划表 |
| 账龄分布 | 拆成回款计划或应收账龄表 |
| 税率 | 合同/预算层保留，多税率拆行 |

## 8. 第一版落地建议

第一版不要追求所有字段一次到位，建议按“能管项目、能看经营、能穿透风险”落地。

### 必做

- 客户主表。
- 项目主表。
- 项目阶段里程碑表。
- 月度经营计划表。
- 月度经营实际表。
- 风险问题表。
- Excel 导入与校验。

### 可后置

- 需求与范围表。
- 变更与验收表。
- 资源成本到人。
- 供应商绩效。
- 自动审批流。

### 建议填报节奏

- 项目立项时：维护项目主表、预算、初始里程碑。
- 每月月初：项目经理维护月度经营计划、回款预测、资源计划。
- 每周：维护里程碑进度、风险问题。
- 每月结账后：经营管理导入实际收入、成本、回款、应收。
- 每月经营分析前：系统生成部门和项目穿透看板。

## 9. 最小可用 Excel 模板

如果暂时仍用 Excel 管理，建议先替换为以下 sheet：

1. `项目主表`
2. `客户主表`
3. `里程碑计划`
4. `月度经营计划`
5. `月度经营实际`
6. `资源计划`
7. `风险问题`
8. `变更验收`
9. `字段字典`

Excel 模板规则：

- 每张表第一列都保留 `项目编号`。
- 客户主表以 `客户编号` 为唯一键，项目主表通过 `客户编号` 关联客户。
- 所有月份统一用 `月份` 字段，不再横向展开 1-12 月。
- 所有状态字段使用下拉字典。
- 实际经营数单独 sheet 管理，导入后锁定。
- 项目主表只保留当前状态和汇总结果，不维护明细。

## 10. 关键变化

这次优化的核心变化是：

- 从“经营结果填报”转为“项目全过程管理”。
- 从“一张宽表”转为“项目主表 + 多张过程明细表”。
- 从“人工填累计数”转为“月度明细自动汇总”。
- 从“只看收入毛利”转为“进度、范围、资源、风险、回款、毛利一起看”。
- 从“客户只是项目字段”转为“客户档案、客户项目、客户合同、客户回款、客户风险一起管”。
- 从“事后统计”转为“过程预警”。
